1.3 Volume-Formatierung und Disk-Konfiguration

⚡
AchtungExchange erfordert 64 KB Allocation Unit Size auf DB- und Log-Volumes. MCDB-SSD (F:) muss RAW/unformatiert bleiben – wird in Phase 5b automatisch konfiguriert.
VolumeZweckRAIDGrösseFilesystemAUSCache
D: DATAExchange Datenbanken (EDB)RAID 1+0 (10 Stripes, 256 KiB Strip)~22 TBReFS64 KB (65536)Controller-Cache
E: LOGSExchange TransaktionslogsRAID 1 (256 KiB Strip)~1.7 TBNTFS64 KB (65536)SSD Smart Path
F: ExchangeMCDBMetaCacheDatabase (SSD Cache)RAID 1 (256 KiB Strip)~894 GBRAW (unformatiert lassen!)—SSD Smart Path
ℹ
HPE Smart Array KonfigurationDie Strip-Grösse von 256 KiB auf dem Controller ist korrekt. Bei RAID 1 ist die Strip-Grösse irrelevant (reiner Mirror). Controller-Cache auf dem grossen RAID 1+0 ist optimal, SSD Smart Path auf den RAID-1 SSDs umgeht den Controller-Cache und reduziert Latenz.
PowerShell – Formatierung prüfen und korrigieren
# Aktuelle Formatierung prüfen
Get-Volume | ft DriveLetter,FileSystemType,AllocationUnitSize,Size

# Ziel-Konfiguration:
# D: (DATA)  → ReFS, 64KB AUS (65536)
# E: (LOGS)  → NTFS, 64KB AUS (65536)
# F: (MCDB)  → RAW lassen (Phase 5b)

# Falls E: mit 4KB formatiert – neu formatieren (löscht Daten!):
Format-Volume -DriveLetter E -FileSystem NTFS `
  -AllocationUnitSize 65536 -NewFileSystemLabel "LOGS" -Confirm:$false

# Falls D: mit 4KB formatiert – neu formatieren (löscht Daten!):
Format-Volume -DriveLetter D -FileSystem ReFS `
  -AllocationUnitSize 65536 -NewFileSystemLabel "DATA" -Confirm:$false

# F: auf RAW zurücksetzen falls bereits formatiert (Disk-Nummer anpassen!):
# Clear-Disk -Number 3 -RemoveData -Confirm:$false

# Controller und Disk-Info prüfen
Get-PhysicalDisk | ft DeviceId,MediaType,BusType,Size,FriendlyName
Get-Disk | ft Number,FriendlyName,PartitionStyle,Size

3.2 Zertifikat importieren – Import-ExchangeCert.ps1

⚡
AchtungDirekt nach den VDirs – kein weiterer Schritt dazwischen.
PowerShell – Import-ExchangeCert.ps1
# Aufruf: .\Import-ExchangeCert.ps1 -Server "NewEX01" -CertPath "\\share\exchange.pfx" -CertPassword "PW"
param(
    [Parameter(Mandatory=$true)] [string]$Server,
    [Parameter(Mandatory=$true)] [string]$CertPath,
    [Parameter(Mandatory=$true)] [string]$CertPassword
)

$secPw = ConvertTo-SecureString $CertPassword -AsPlainText -Force
Write-Host "[$Server] Importiere Zertifikat..." -ForegroundColor Cyan

$thumbprint = (Import-ExchangeCertificate `
  -Server $Server `
  -FileData ([Byte[]](Get-Content $CertPath -Encoding Byte -ReadCount 0)) `
  -Password $secPw `
  -FriendlyName "Exchange SE Cert").Thumbprint

Write-Host "[$Server] Aktiviere Zertifikat..." -ForegroundColor Yellow
Enable-ExchangeCertificate -Server $Server -Thumbprint $thumbprint -Services IIS,SMTP -Force

# Verifikation
Get-ExchangeCertificate -Server $Server | ft Thumbprint,Subject,Services,NotAfter
Write-Host "[$Server] Zertifikat erfolgreich importiert und aktiviert." -ForegroundColor Green

3.3 Health Check – Test-NewExchangeServer.ps1

PowerShell – Test-NewExchangeServer.ps1
# Aufruf: .\Test-NewExchangeServer.ps1 -Server "NewEX01"
param([Parameter(Mandatory=$true)] [string]$Server)

$fqdn = "$Server.DOMAIN.intern"
$errors = @()
Write-Host "=== Health Check: $Server ===" -ForegroundColor Cyan

# 1. Build Version
Write-Host "Build: $((Get-ExchangeServer $Server).AdminDisplayVersion)"

# 2. Dienste
$stopped = Get-Service -ComputerName $Server MSExchange* | where Status -ne "Running"
if ($stopped) { $errors += "Gestoppte Dienste"; Write-Host "FEHLER: Dienste nicht gestartet" -ForegroundColor Red }
else { Write-Host "OK: Alle Exchange Dienste laufen" -ForegroundColor Green }

# 3. Service Health
$unhealthy = Test-ServiceHealth -Server $Server | where { $_.RequiredServicesRunning -eq $false }
if ($unhealthy) { $errors += "Unhealthy Services"; Write-Host "FEHLER: Service Health" -ForegroundColor Red }
else { Write-Host "OK: Service Health" -ForegroundColor Green }

# 4. OWA Health Check
try {
  $r = Invoke-WebRequest "https://$fqdn/owa/healthcheck.htm" -UseBasicParsing -TimeoutSec 10
  if ($r.StatusCode -eq 200) { Write-Host "OK: OWA Healthcheck 200" -ForegroundColor Green }
  else { $errors += "OWA $($r.StatusCode)"; Write-Host "FEHLER: OWA $($r.StatusCode)" -ForegroundColor Red }
} catch { $errors += "OWA nicht erreichbar"; Write-Host "FEHLER: OWA nicht erreichbar" -ForegroundColor Red }

# 5. MAPI Connectivity
try { Test-MAPIConnectivity -Server $Server | ft DisplayName,Result; Write-Host "OK: MAPI" -ForegroundColor Green }
catch { $errors += "MAPI Fehler" }

# 6. VDir URL prüfen
$url = (Get-OwaVirtualDirectory -Server $Server).InternalUrl
if ($url -like "*mail.DOMAIN.intern*") { Write-Host "OK: OWA URL: $url" -ForegroundColor Green }
else { $errors += "OWA URL falsch: $url"; Write-Host "FEHLER: OWA URL: $url" -ForegroundColor Red }

# 7. Zertifikat
$cert = Get-ExchangeCertificate -Server $Server | where { $_.Services -match "IIS" }
if ($cert) { Write-Host "OK: Zertifikat: $($cert.Subject)" -ForegroundColor Green }
else { $errors += "Kein IIS-Zertifikat"; Write-Host "FEHLER: Kein IIS-Zertifikat" -ForegroundColor Red }

# Ergebnis
Write-Host ""
if ($errors.Count -eq 0) { Write-Host "=== $Server: ALLE CHECKS OK – weiter mit naechstem Server ===" -ForegroundColor Green }
else { Write-Host "=== $Server: $($errors.Count) FEHLER – NICHT weiter! ===" -ForegroundColor Red
  $errors | ForEach { Write-Host "  - $_" -ForegroundColor Red } }

Phase5b

MetaCacheDatabase (MCDB)

SSD-basierte Beschleunigung für DAG – F: Volume auf jedem Server

ℹ
InfoMCDB beschleunigt den Zugriff auf häufig genutzte Mailbox-Daten über SSDs. Voraussetzungen: DAG muss healthy sein, Datenbanken müssen erstellt und gemountet sein. Alle 4 Server benötigen identische SSDs (gleiche Grösse und Typ). MCDB-SSD (F:) muss unformatiert/RAW sein – das Manage-MCDB-Skript übernimmt die Formatierung.
⚡
Achtung – F: Volume vorbereitenFalls F: bereits formatiert ist (wie bei euch aktuell ReFS 4KB), muss das Volume zuerst auf RAW zurückgesetzt werden. Das Manage-MCDB-Skript erwartet eine unformatierte SSD.

5b.1 SSD auf RAW zurücksetzen (falls bereits formatiert)

Auf JEDEM der 4 Server ausführen. ACHTUNG: Löscht alle Daten auf F:!

PowerShell – Auf jedem Server
# Disk-Nummer der MCDB-SSD ermitteln
Get-Disk | ft Number,FriendlyName,Size,PartitionStyle

# F: Volume auf RAW zurücksetzen (Disk-Nummer anpassen!)
Clear-Disk -Number 3 -RemoveData -Confirm:$false

# Prüfen: Disk muss jetzt RAW sein
Get-Disk 3 | fl Number,PartitionStyle,Size
# PartitionStyle muss "RAW" zeigen

5b.2 DAG-Parameter für MCDB setzen

Einmalig ausführen – setzt die AutoDag-Werte die MCDB für die Grössenberechnung benötigt.

PowerShell
# DAG-Parameter anpassen (Anzahl DBs, Kopien, Server)
Set-DatabaseAvailabilityGroup "DAG-New" `
  -AutoDagTotalNumberOfDatabases 4 `
  -AutoDagDatabaseCopiesPerDatabase 4 `
  -AutoDagTotalNumberOfServers 4

# Verifikation
Get-DatabaseAvailabilityGroup "DAG-New" | fl AutoDag*

5b.3 Manage-MCDB Modul laden und Prerequisites konfigurieren

ℹ
InfoDas Manage-MCDB CMDLet ist nicht standardmässig verfügbar – es muss in jeder EMS-Session manuell importiert werden. Nach ConfigureMCDBPrerequisite bis zu 20 Minuten auf AD-Replikation warten.
PowerShell

Sequenziell – ein Server nach dem anderen. Dazwischen warten bis CopyQueue und ReplayQueue auf 0 sind.

5b.7 Fallback: Manuelle SSD-Formatierung

ℹ

PowerShell – Fallback: Manuelle Formatierung
# Verzeichnisse anlegen
New-Item -Path "C:\ExchangeMCDBVolumes" -ItemType Directory -Force
New-Item -Path "C:\ExchangeMetaCacheDbs" -ItemType Directory -Force

# SSD manuell als ReFS formatieren (Integrity Streams deaktivieren!)
Get-Disk 3 | Initialize-Disk -PartitionStyle GPT -PassThru |
  New-Partition -UseMaximumSize |
  Format-Volume -FileSystem ReFS -NewFileSystemLabel "ExchangeMCDB" `
    -SetIntegrityStreams $false

# Volume in Mount Point einhängen
# (Drive Letter entfernen, stattdessen Mount Point nutzen)
Add-PartitionAccessPath -DiskNumber 3 -PartitionNumber 2 `
  -AccessPath "C:\ExchangeMCDBVolumes\ExchangeSSD1"

# Dann ConfigureMCDBOnServer erneut ausführen
Manage-MCDB -DagName "DAG-New" `
  -ConfigureMCDBOnServer `
  -ServerName "NewEX01" `
  -SSDSizeInBytes 960160000000 -Verbose

7.2 Migration Batch erstellen – New-MigrationBatch-FromCSV.ps1

PowerShell – New-MigrationBatch-FromCSV.ps1
# CSV-Format: eine Spalte "EmailAddress"
# Aufruf: .\New-MigrationBatch-FromCSV.ps1 -BatchName "Batch-01" -CsvPath "C:\temp\batch01.csv" -TargetDatabase "DB-New-01"
param(
    [Parameter(Mandatory=$true)] [string]$BatchName,
    [Parameter(Mandatory=$true)] [string]$CsvPath,
    [Parameter(Mandatory=$true)] [string]$TargetDatabase
)

New-MigrationBatch `
  -Name $BatchName `
  -CSVData ([System.IO.File]::ReadAllBytes($CsvPath)) `
  -TargetDatabases $TargetDatabase `
  -AutoStart `
  -AutoComplete:$false

Write-Host "Batch $BatchName gestartet:" -ForegroundColor Green
Get-MigrationBatch -Identity $BatchName | ft Name,Status,TotalCount
